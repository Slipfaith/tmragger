# tmragger

Инструмент для подготовки TMX-файлов трансляционной памяти к использованию в RAG-системах.

## Что делает

Читает `.tmx`-файл, анализирует каждый TU (translation unit) и предлагает набор правок:

- **Сплит сегментов** — делит TU на несколько, если source и target содержат несколько предложений и выравниваются корректно
- **Очистка пробелов** — убирает двойные пробелы и пробелы по краям
- **Удаление служебной разметки** — inline-теги, игровой markup (`^{...}^`, `$m(...|...)`), `%...%`-токены
- **Удаление мусорных TU** — пустые, числовые, декоративные сегменты
- **Удаление дублей TU** — убирает повторяющиеся пары (source, target), оставляя первое вхождение
- **Диагностика WARN** — помечает подозрительные TU (разница длины, несовпадение языка, идентичные src/tgt) без удаления
- **Gemini verification** *(опционально)* — проверяет решения о сплите через Gemini API

## Рабочий процесс

Обработка двухфазная: сначала приложение строит **план** (список предложений), затем пользователь просматривает его в диалоге ревью и запускает **применение** только принятых правок.

```
Файлы → [Plan] → Ревью предложений → [Apply] → Готовый TMX + отчёты
```

## Запуск

### GUI

```bash
python main.py
```

Перетащите `.tmx`-файлы в окно или нажмите кнопку добавления. Настройте этапы в панели слева, нажмите **Run**.

### CLI

```bash
python main.py --cli --input file.tmx --output-dir ./output
```

Основные флаги:

| Флаг | Описание |
|---|---|
| `--input <path...>` | Один или несколько входных TMX |
| `--output-dir <dir>` | Директория для результатов |
| `--no-split` | Отключить сплит |
| `--no-cleanup-spaces` | Отключить очистку пробелов |
| `--no-cleanup-garbage` | Отключить удаление мусорных TU |
| `--no-cleanup-warnings` | Отключить диагностику WARN |
| `--verify-gemini` | Включить проверку сплита через Gemini |
| `--gemini-api-key <key>` | API-ключ Gemini (или `GEMINI_API_KEY` env) |
| `--dry-run` | Анализ без записи файла |
| `--report-dir <dir>` | Директория для JSON-отчётов |
| `--html-report-dir <dir>` | Директория для HTML diff-отчётов |
| `--xlsx-report-dir <dir>` | Директория для XLSX-отчётов |

## Отчёты

После обработки генерируются три вида отчётов в `tmx-reports/`:

- **JSON** — машиночитаемый лог всех правок и статистика
- **HTML** — визуальный diff: было / стало для каждого TU
- **XLSX** — многолистовой отчёт: сводка, сплиты, очистка, предупреждения

## Gemini verification

Включается чекбоксом в GUI или флагом `--verify-gemini` в CLI. Требует API-ключ.

При включении каждый кандидат на сплит отправляется в Gemini, который возвращает вердикт `OK / WARN / FAIL` и уровень уверенности. Решения кэшируются внутри прогона (одинаковые кандидаты проверяются один раз) и могут сохраняться между прогонами (`--gemini-cache-file`).

Дублирующиеся TU при включённом удалении дублей пропускаются до проверки Gemini — это снижает количество API-вызовов и стоимость.
